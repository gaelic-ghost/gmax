# v0.0.8 Release Notes

`v0.0.8` is the terminal-history restoration follow-through release for `gmax`.

It builds on the `v0.0.7` command-and-sizing work by making ordinary shell
scrollback actually come back on relaunch instead of stopping at command
history and launch context alone.

## What Changed

- fixed the ordinary live relaunch path so terminal sessions restore from
  persisted pane-session snapshots instead of rebuilding shells from layout
  alone
- moved terminal-history capture toward the real shell stream by preferring raw
  host output over the older screen-shaped text export path
- delayed shell launch until the embedded terminal view has a real viewport, so
  restored history is not wiped out by early resize/setup churn
- kept restored-history metadata for ordinary scroll position and
  alternate-buffer detection in place, while leaving full alternate-buffer and
  TUI replay as future work
- aligned the maintainer docs and release checklist with the shipped relaunch
  restore behavior

## Known Gaps

- full alternate-buffer and emulator-state restoration are still backlog work,
  not shipped behavior
- saved-window reopen behavior is still primarily covered at the persistence
  level rather than by a dedicated end-to-end UI automation
- the broader accessibility and multi-window manual audit for the `v0.1.0`
  quality bar is still ahead
