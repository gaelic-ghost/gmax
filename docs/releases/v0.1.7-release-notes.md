# v0.1.7 Release Notes

`v0.1.7` is a split-pane resize stability release for `gmax`.

It builds on the packaged DMG distribution work in `v0.1.6` by making
recursive pane resizing usable with real terminal panes instead of letting live
SwiftTerm reflow dominate the drag interaction.

## What Changed

- fixed split-divider drag math so resizing starts from the split fraction at
  the beginning of the drag instead of remeasuring against the moving divider
  handle
- stopped publishing every intermediate drag fraction through the workspace
  model and persistence scheduler
- changed pointer resizing to show a live preview divider while dragging, then
  commit the final split fraction on mouse-up so terminal panes reflow once
- updated the Quick Start to point at the packaged DMG release path

## Known Gaps

- pane contents do not resize live while the mouse is down; the preview divider
  shows the pending split and the real pane layout updates when the drag ends
- split dividers still clamp at the minimum pane size, so the cursor can keep
  moving after the divider reaches the smallest allowed pane width or height
- richer divider affordances such as keyboard nudging, snap points, or a more
  polished preview treatment remain future product work
