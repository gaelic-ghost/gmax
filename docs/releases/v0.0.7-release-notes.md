# v0.0.7 Release Notes

`v0.0.7` is the command-and-window-geometry follow-through release for `gmax`.

It builds on the `v0.0.6` mixed-library and restoration work by making the keyboard surface more intentional and by relaxing a few window and sheet sizing constraints that were making the app feel heavier than it needed to.

## What Changed

- aligned the main workspace and window shortcuts around a cleaner split:
  - `Command-N` for `New Workspace`
  - `Shift-Command-N` for `New gmax Window`
  - `Option-Command-O` for `Open Recent Workspace`
  - `Shift-Command-O` for `Open Recent Window`
  - `Option-Command-S` for `Close Workspace to Library`
  - `Shift-Command-S` for `Close Window to Library`
  - `Option-Command-W` for `Close Workspace`
  - `Shift-Command-W` for `Close Window`
  - `Option-Command-R` for `Rename Workspace`
- changed the sidebar and inspector visibility commands to intentional `Command-B` and `Shift-Command-B` shortcuts while preserving the real `Show ...` / `Hide ...` menu-title flip based on scene state
- renamed the old undo-close wording to `Open Recent Workspace` and `Open Recent Window` across the active command and maintainer surfaces
- added a Settings toggle for `Auto-save closed workspaces and windows`, so explicit close-to-library commands remain available even when automatic archival is off
- made automatic window archival respect that setting for closed windows, while keeping explicit `Close Window to Library` behavior intact
- reduced the default new-window size and switched the main split-view columns and library sheet to ideal-size-biased sizing instead of the previous heavier minimum-driven constraints

## Known Gaps

- ordinary live app relaunch still does not rehydrate terminal scrollback into the active SwiftTerm surface
- saved-window reopen behavior is covered at the persistence level, but the dedicated UI automation for that path is still future work
- the broader accessibility and multi-window manual audit for the `v0.1.0` quality bar is still ahead
