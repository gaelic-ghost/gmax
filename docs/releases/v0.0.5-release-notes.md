# v0.0.5 Release Notes

`v0.0.5` is the persistence-foundation release for `gmax`.

It turns the earlier shell prototype into a much more coherent multi-window app:

- workspace identity is now stable across live and saved-library flows
- each workspace window has durable Core Data-backed window records
- recent workspace reopen behavior is durable and window-local instead of an in-memory-only stack
- workspace save and restore behavior now follows the intentional multi-window model throughout the maintainer docs and tests
- explicit `Close Window` and `Undo Close Window` commands now exist alongside the adaptive `Command-W` path

## What Changed

- unified live and library workspace identity around one durable `WorkspaceID`
- moved launch-restore and closed-window durability onto `WorkspaceWindowEntity`
- moved per-window recent workspace history onto `WindowWorkspaceMembershipEntity` plus workspace recency timestamps
- hardened persistence saves around close, quit, activation changes, and periodic background saves
- expanded unit and UI coverage around focus, close, split, persistence, and restoration behavior
- aligned maintainer docs with the shipped multi-window persistence and command model

## Known Gaps

- ordinary live app relaunch still does not rehydrate terminal scrollback into the active SwiftTerm surface
- the unified library still models saved workspaces only; saved windows are planned follow-through work
- the broader accessibility and multi-window manual audit for the `v0.1.0` quality bar is still ahead
