# v0.1.0 Release Notes

`v0.1.0` is the first shell-integration checkpoint release for `gmax`.

It builds on the `v0.0.9` browser-pane work by giving ordinary terminal panes a
small semantic model for prompt, command, bell, and notification state instead
of treating everything as undifferentiated terminal output.

## What Changed

- added a lightweight terminal host-event parser for prompt start, command
  start, command finish, bell, and explicit terminal notification signals
- added a first real `zsh` shell-integration path that emits prompt, command,
  exit-status, and cwd markers without replacing the user's existing shell
  startup flow
- added pane-local shell-status chrome for running, finished, failed, and
  bell-attention state, plus workspace-sidebar bell-count badges and per-window
  bell aggregation for future follow-through
- added terminal inspector metadata for bell count, last bell time, and the
  most recent explicit terminal notification
- updated the maintainer notes and roadmap to treat further shell integrations,
  `bash` support, and macOS notification routing as the next follow-through
  steps instead of leaving them implicit

## Known Gaps

- the shipped shell-integration emission path currently covers `zsh`, not
  `bash` or other shells yet
- explicit terminal notifications are captured in pane state and the inspector,
  but they do not yet route into real macOS notifications
- the current shell-aware state is runtime metadata only, not durable command
  history or richer restored shell semantics
